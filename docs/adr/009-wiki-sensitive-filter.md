# ADR-009: Wiki (document.page) и фильтр чувствительного контента

## Статус

Accepted

## Контекст

Команда запросила поддержку wiki (модуль `document.page` / EDMS) в MCP-сервере.
Wiki-страницы часто содержат пароли, ключи доступа и другую конфиденциальную
информацию. Даже в полностью доверительном режиме (`ODOO_TRUST_MODE=full`)
утечка таких данных через AI-агента недопустима.

## Рассмотренные варианты

### Конфигурация фильтра

1. **Hardcoded + ENV override паттернов** — безопасно из коробки,
   но regex в переменных окружения неудобен.
2. **Конфиг-файл** (`.odoo-sensitive-filter.yml`) — гибко,
   но лишний файл, который можно случайно удалить.
3. **Hardcoded в коде, только on/off** — максимально безопасный,
   паттерны нельзя ослабить случайно.

### Матрица состояний

| # | TrustMode  | Чувствительный контент | Результат                                          |
|---|------------|------------------------|----------------------------------------------------|
| 1 | restricted | нет                    | Метаданные: заголовок, дата, категория, ссылка      |
| 2 | restricted | да                     | То же (тело никогда не показывается)                |
| 3 | full       | нет                    | Всё: заголовок, тело, история                       |
| 4 | full       | да                     | Метаданные + причина цензуры. Тело скрыто полностью |

## Решение

Выбран **вариант 3**: паттерны зашиты в коде, без кастомизации.

- Паттерны: `password`, `passwords`, `пароль`, `пароли`, `парол`,
  `доступ`, `credentials`, `secret`, `token`, `api.key`, `ssh.key`,
  `private.key`
- Переключатель: `ODOO_WIKI_SENSITIVE_FILTER=off` — единственный
  способ отключить (с предупреждением в логах)
- Фильтр проверяет **название + содержимое** страницы
- При срабатывании — **вся страница** цензурируется без исключений
- Фильтр размещён в `domain/sensitive.py` как чистая функция

## Последствия

- **Положительные**: безопасно по умолчанию, нельзя случайно ослабить,
  тестируемо без инфраструктурных зависимостей
- **Отрицательные**: нельзя добавить кастомные паттерны без изменения кода
- **Риски**: false positive — страница с упоминанием "секретный ингредиент"
  будет заблокирована. Допустимый компромисс: безопасность важнее доступности
