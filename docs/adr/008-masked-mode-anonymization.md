# ADR-008: Masked mode — анонимизация корпоративных данных

## Статус

Accepted

## Контекст

При использовании внешнего AI-вендора (Claude, GPT и пр.) корпоративные
данные передаются в контекст модели. Даже в full mode это создаёт риск
утечки: имена сотрудников, названия компаний, проектов, email-адреса,
содержание переписки.

Restricted mode решает проблему радикально — не показывает ничего.
Но это снижает полезность AI. Нужен промежуточный вариант: AI видит
**структуру и связи**, но не видит **реальные значения**.

## Решение

Добавить третий режим доступа — **masked**.

### Три уровня доверия

| Режим        | ENV                       | Что видит AI                           |
|--------------|---------------------------|----------------------------------------|
| `restricted` | По умолчанию              | Только ID, стадия, часы, дедлайн       |
| `masked`     | `ODOO_TRUST_MODE=masked`  | Структура + анонимизированные значения  |
| `full`       | `ODOO_TRUST_MODE=full`    | Все данные как есть                    |

### Что маскируется

| Категория       | Пример оригинала                     | Маскированное значение   |
|-----------------|--------------------------------------|--------------------------|
| Имена           | Alexei Petrov                        | [Сотрудник ID: 33]       |
| Компании        | OOO «SolarTech»                      | [Компания ID: 906]       |
| Email           | a.petrov@solartech.ru                | (скрыт)                  |
| Проекты         | Project Mercury                      | [Проект ID: 99]          |
| Названия задач  | Implement CI/CD pipeline for staging | [Задача ID: 1228]        |
| Описания задач  | \<html содержимое\>                  | (содержимое скрыто)      |
| Комментарии     | Items 3 and 4 need clarification     | (содержимое скрыто)      |
| Теги            | P1-Do now                            | [Тег ID: 50]             |
| Вехи            | Sprint 14                            | [Веха ID: 12]            |

### Формат псевдонимов

Формат `[Тип ID: N]` вместо нумерованных псевдонимов:
- Сохраняет привязку к реальному Odoo ID
- AI может использовать ID для операций (assignee_ids, project_id)
- Не требует маппинга в памяти — ID является маппингом
- Консистентно между запросами и перезапусками

### Архитектура

```
Odoo ERP
    ↑
   ACL (transport → protocol → mapper)
    ↑
Service Layer
    ↑
Masking Layer ← замена полей по категориям
    ↑
MCP Server / CLI
```

Masking Layer — тонкая прослойка между Service Layer и представлением.
Работает на уровне доменных объектов (Task, User, Project), не на
уровне Odoo-записей.

### Реализация

```python
class DataMasker:
    def mask_user(self, user: User) -> User:
        return User(id=user.id, name=f"[Сотрудник ID: {user.id}]")

    def mask_project(self, project: Project) -> Project:
        return Project(id=project.id, name=f"[Проект ID: {project.id}]")

    def mask_task(self, task: Task) -> Task:
        return task.model_copy(update={
            "name": f"[Задача ID: {task.id}]",
            "description": None,
            "assignees": [self.mask_user(u) for u in task.assignees],
            "partner": self._mask_partner(task.partner),
            "project": self.mask_project(task.project),
            "tags": [self._mask_tag(t) for t in task.tags],
            "messages": [],
            "attachments": [],
        })
```

### Что НЕ маскируется

- ID записей (нужны для операций: update_task, change_stage)
- Стадии (Новое, В разработке — это workflow, не sensitive)
- Числовые значения (часы, прогресс, даты)
- Приоритеты и статусы
- Подзадачи (маскируются рекурсивно по тем же правилам)

## Последствия

- **Положительные**:
  - AI работает со структурой без утечки корпоративных данных
  - ID-based псевдонимы не требуют маппинга в памяти
  - Подходит для демонстраций, аудита, внешних вендоров

- **Отрицательные**:
  - AI не может помочь с содержанием задач (описания скрыты)
  - Дополнительная сложность в коде

- **Риски**:
  - AI может запросить раскрытие ("Кто Сотрудник ID: 33?")
  - Митигация: system prompt MCP-сервера объясняет masked mode
